# Harbor — Agent Guidelines

## Project Overview

Harbor is a personal macOS/Windows desktop app (Tauri v2) that turns your machine into a VLESS WebSocket exit node through Cloudflare Tunnel. It bundles `sing-box` and `cloudflared` binaries, generates VLESS links with QR codes, and manages both processes with real-time log streaming.

## Tech Stack

| Layer | Technology |
|-------|-----------|
| Desktop framework | Tauri v2 |
| Backend | Rust (edition 2021) |
| Frontend | TypeScript (ES2022), vanilla DOM — no framework |
| Build | Vite 7, TypeScript 5 |
| Test | Vitest 4 (frontend + script tests) |
| Bundled binaries | sing-box 1.13.11, cloudflared 2026.3.0 |

## Key Commands

```bash
npm run dev              # Tauri dev with hot-reload
npm run build            # TypeScript + Vite production build
npm run test             # Run all tests (frontend + script .test.mjs files)
npm run bump             # Bump version + validate changelog
npm run validate:changelog  # Check current version has changelog entry
cargo test --lib --manifest-path src-tauri/Cargo.toml  # Rust tests
```

## Architecture

- **Frontend** (`src/`): Vanilla TypeScript with innerHTML rendering. No React/Vue. State is module-level variables. UI re-renders via `render()` function that sets `app.innerHTML`.
- **Backend** (`src-tauri/src/`): Rust with Tauri commands. Manages sing-box and cloudflared child processes via `std::process::Command`. Uses process groups for cleanup.
- **Communication**: Tauri `invoke()` for commands, `listen()` for events (logs, tray actions).
- **Settings**: Persisted in `localStorage`; saved user values override bundled defaults.
- **Bundled defaults**: `harbor-defaults.defaults.json` is tracked; `harbor-defaults.bundle.json` is gitignored and generated by `prepare-bundled-defaults.mjs`.

## Important Patterns

- **No framework frontend**: UI is rendered via template strings in `main.ts`. Always read `main.ts` before modifying UI.
- **State is global**: Module-level `settings`, `status`, `preview`, `logs` variables. No state management library.
- **Process management**: Rust spawns child processes with process groups (`libc::setpgid`). Stop/quit kills the entire group.
- **Token security**: Tunnel token passed via `TUNNEL_TOKEN` env var, never in CLI args. User-entered tokens are persisted locally to override bundled defaults.
- **Update system**: External API at `harbor.timwuhaotian.dev` — version check and token-verified download. Do not modify these URLs without coordinating backend changes.
- **CI/CD**: GitHub Actions builds macOS (signed) and Windows. Version detected from `package.json`. Release blocked without changelog entry.

## File Conventions

- Script tests: `scripts/*.test.mjs` — uses local `test()` runner, not Vitest
- Frontend tests: `src/*.test.ts` — Vitest
- Rust tests: inline `#[cfg(test)]` modules in `src-tauri/src/*.rs`
- i18n: `src/i18n.ts` — simple dictionary lookup, EN default
- Version: injected as `__APP_VERSION__` via Vite define (see `vite.config.ts`)

## Before Making Changes

1. Read the relevant source file first — never guess the architecture
2. Run `npm test` after frontend changes
3. Run `cargo test --lib --manifest-path src-tauri/Cargo.toml` after Rust changes
4. Check `CHANGELOG.md` follows Keep a Changelog format
5. Update i18n keys in both `en` and `zh-CN` dictionaries if adding UI text
6. Do NOT commit `.env`, `harbor-defaults.bundle.json`, or bundled binaries
